# シフト提出アプリ

## 作成物の説明

アルバイトスタッフのシフト希望提出と、店長によるシフト作成・公開を行うWebアプリを作成した。

従来、紙やメッセージで集めていたシフト希望は、提出状況の確認、転記、割当、公開後の共有に手間がかかりやすい。そこで、スタッフが働ける日付と時間帯を画面上で提出し、店長がその内容を見ながらシフトを作成できる仕組みを開発した。

このアプリでは、スタッフ、店長、管理者の役割ごとに画面を分け、シフト希望の提出、割当、公開状態の管理、ユーザー管理、コスト確認を行えるようにしている。

## 主な機能

- スタッフが月ごとにシフト希望を提出できる
- 日付と時間帯を選んで、入れる時間を登録できる
- 店長がスタッフの提出状況を確認できる
- 店長が希望提出を見ながらシフトを手動で作成できる
- 公開されたシフトをスタッフが確認できる
- 管理者がユーザー、提出データ、割当データを確認できる
- 管理画面でAWS利用コストの実績とリアルタイム推定額を確認できる
- WAFを無効化してコストを抑えつつ、必要時には再度有効化できる

## 自身が担当した役割

要件整理から設計、実装、デプロイ、改善までを担当した。

主な担当工程は以下の通り。

- シフト提出、シフト作成、シフト確認画面の設計と実装
- スタッフ、店長、管理者の権限ごとの画面分岐
- 管理画面の実装
- 提出データ、割当データ、公開状態の保存・取得処理
- ログイン、ログアウト、認証後遷移の調整
- AWS上へのデプロイ
- コスト監視画面の追加
- WAF無効化と再有効化しやすい構成への変更
- README、仕様書などのドキュメント整備

ReactやTypeScriptなど、知識が不十分な部分はAIを活用したバイブコーディングで実装し、生成されたコードを確認・修正しながら完成させた。

## 直面した課題と解決方法

### ログアウト後にログイン画面へ戻れない問題

ログアウト後の遷移先URLがCognito側の設定と一致しておらず、Invalid requestが表示される問題が発生した。

解決方法として、CognitoのログアウトURLとアプリ側のリダイレクト先を見直し、ログアウト後に正しい画面へ戻るように修正した。

### 管理者画面でデータが見えづらい問題

管理者画面ではユーザーIDだけが表示され、どのスタッフのデータなのか判別しづらい状態だった。また、シフト提出や割当の一覧も管理しにくかった。

解決方法として、ユーザー名やメールアドレスを表示し、提出、割当、公開状態、ユーザー管理を管理画面から確認しやすいように改善した。

### 店長画面と管理者画面でシフトデータの見え方が違う問題

バイト・店長画面では見えているシフトが、管理者画面では確認しづらい状態だった。

解決方法として、管理者用APIと画面を実データに接続し、提出データ、割当データ、公開状態を管理者画面から確認できるようにした。

### コストが高くなりやすい問題

AWS WAFの固定費により、20人程度の利用規模に対して月額コストが高くなりやすいことが分かった。

解決方法として、WAFを無効化して固定費を削減した。その代わり、Cognito認証、Cognitoのログイン失敗ロック、API Gatewayのスロットリング、Budget通知で最低限の防御を行う構成にした。また、必要になった場合は`enableWaf=true`でWAFを再度有効化できるようにした。

### コストをリアルタイムに把握したい問題

AWS Cost Explorerは請求データの反映に遅延があるため、現在の利用状況をすぐに確認しにくい。

解決方法として、Cost Explorerによる実績額に加え、CloudWatchメトリクスから利用量を取得して概算料金を計算するリアルタイム推定機能を追加した。円換算には毎日の為替レートを利用し、ブラウザ側で取得・キャッシュすることで追加のAWSコストを抑えた。

### 時間グリッドが分かりづらい問題

時間ラベルと選択マスが横に分かれており、どの時間帯を選んでいるのか分かりにくかった。

解決方法として、各マスに`09:00-09:30`のような時間帯を直接表示し、選択中の時間が直感的に分かるようにした。

## 技術情報

### 使用モデル

AIモデルをアプリ本体の機能としては使用していない。

ただし、開発時にはReact、TypeScript、AWS構成の実装補助としてAIを活用した。AIが生成したコードをそのまま使うのではなく、動作確認、型チェック、デプロイ確認を行いながら修正した。

### フロントエンド

- React
- TypeScript
- Vite
- React Router
- react-oidc-context

スタッフ向け画面と管理者向け画面を分けて実装した。スタッフ向け画面では、シフト希望提出、シフト確認、店長向けシフト作成を扱う。管理者向け画面では、提出データ、割当データ、公開状態、ユーザー、コストを確認できる。

### API

- API Gateway
- Lambda
- Cognito Authorizer

主要APIは以下の通り。

- `GET /availability`
- `POST /availability`
- `GET /assignments`
- `POST /assignments`
- `GET /publish`
- `POST /publish`
- `GET /admin/availability`
- `POST /admin/availability`
- `GET /admin/assignments`
- `POST /admin/assignments`
- `GET /admin/users`
- `POST /admin/users`
- `GET /admin/cost`
- `POST /admin/ttl`

### データベース

- DynamoDB

月単位のデータを中心に、提出データ、割当データ、公開状態を保存している。TTLを設定し、古いデータを整理できるようにしている。

### 認証・認可

- Amazon Cognito

ユーザーはCognitoで認証する。管理者はCognitoグループで判定し、管理画面や管理APIにアクセスできるユーザーを制限している。ログイン失敗時はCognito標準の一時ロックアウトにより、総当たり攻撃への対策を行っている。

### アーキテクチャ

```text
利用者
  ↓
CloudFront
  ↓
S3 静的サイト

画面操作
  ↓
API Gateway
  ↓
Lambda
  ↓
DynamoDB

認証
  ↓
Cognito
```

管理者向けには別の管理画面を用意し、同じAPIを管理者権限で操作する構成にしている。

### コスト監視

- Cost Explorer
- CloudWatch
- AWS Budgets

Cost Explorerで請求実績を取得し、CloudWatchメトリクスからリアルタイム推定額を計算する。AWS Budgetsで予算通知を設定し、想定外のコスト増加を検知できるようにしている。

### セキュリティ方針

現在はAWS WAFを無効化している。20人程度の小規模利用を想定し、コスト削減を優先したためである。

代わりに、以下の対策を行っている。

- Cognito認証
- Cognitoのログイン失敗ロック
- 管理者グループ制限
- API Gatewayスロットリング
- S3の非公開化
- IAM権限の制限
- Budget通知

必要になった場合は、CDKの`enableWaf=true`を指定してWAFを再度有効化できる。
